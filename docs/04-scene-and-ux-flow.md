# 04. 씬 구성 및 UX 플로우

## 1. 씬 목록

| 씬 | 역할 | 활성 시스템 |
|---|---|---|
| `MainMenu` | 시작 화면, 시나리오 선택(MVP는 1개), 설정, 시뮬레이션 모드 선택 | UI only |
| `Briefing` | 상황·처방 확인, 학습 목표 안내, "시작" 버튼 | UI + 짧은 나레이션 |
| `Simulation_IMInjection` | 메인 플레이 씬 (처치실) | Runner + Interaction + 3D 환경 |
| `Debriefing` | 결과 리포트, 재도전 | UI + ScoreTracker 읽기 |

씬 전환은 `SceneLoader`(싱글톤 or 정적 서비스)가 페이드 인/아웃과 함께 처리.

## 2. 글로벌 UX 플로우

```
[MainMenu]
   │  시작 (기본값: Basic Simulation Mode)
   ▼
[Briefing]  ─(ESC: 메뉴)
   │  확인·시작
   ▼
[Simulation_IMInjection]
   │  절차 완료(13단계) 또는 "포기"
   ▼
[Debriefing]
   │  재도전 ──► [Briefing] (진행 상태 리셋)
   │  메인으로 ──► [MainMenu]
```

## 2.1 시뮬레이션 모드

### Basic Simulation Mode (기본값)

- MainMenu에서 바로 시작되는 표준 학습 모드다.
- 사용자는 현재 단계의 카드, 체크리스트, 선택지, 액션 버튼을 누른다.
- 손 모델은 선택한 액션에 맞는 애니메이션을 재생하고, 3D 환경은 결과 상태만 보여준다.
- 채점은 손 조작 숙련도가 아니라 KABONE 절차 선택, 순서, 확인 여부를 기준으로 한다.
- 간호대 학부생의 예습/복습 경로는 이 모드에서 끝까지 완주 가능해야 한다.

### Hard Hand Simulator Mode (설정에서 ON)

- 기존 `A/S/D/F/Space`, `Z/X/C/V/B`, `RMB`, `LMB`, `Mouse Wheel`, `Left Shift` 기반 손/팔 조작 모드다.
- 기본값은 OFF이며, 고급 연습, 시연, 디버그 용도로 둔다.
- Hard Hand 입력은 `HandActionEvents`로 변환되어 기존 step controller에 전달된다.
- Hard Hand 전용 임상 규칙이나 별도 채점 기준을 만들지 않는다. KABONE 절차 기준은 Basic 모드와 동일하다.

### 3.3.1 Step 1 정보 카드

Step 1 처방 확인은 첫 학습 장벽을 낮추기 위해 카드형 정보 패널로 만든다.

- **처방 카드**: 약명, 용량, 투여경로, 투여시간.
- **환자 카드**: 이름, 등록번호, 병실/상황 정보.
- **투약원칙 카드**: 대상자 등록번호, 대상자명, 약명, 용량, 투여경로, 시간.
- **핵심항목 표시**: KABONE 핵심항목(★)임을 명확히 표시하되, 화면 문구는 학생 친화적으로 작성한다.
- **완료 조건**: 6개 확인 항목 체크 + "확인 완료" 버튼. 누락 시 같은 카드 안에서 즉시 보정하게 한다.

## 4. 카메라 설계 (Simulation 씬)

- **기본 카메라 (Cinemachine Virtual Camera #1)**: 처치실 고정 3인칭. 트레이·환자·학생 포지션이 모두 보이는 앵글.
- **상호작용 줌 카메라 (#2)**: 선택된 3D 도구/부위로 클로즈업. `CinemachineBrain`이 부드럽게 블렌드.
- **환자 부위 카메라 (#3)**: 부위 선정·랜드마크 단계에서 환자 해당 부위로 전환.
- 우클릭 드래그로 현재 가상 카메라의 `POV` 오빗(±15°)만 허용 — 자유시점은 UX 복잡도 증가로 제외.

## 5. 인터랙션 규칙

1. **Basic 액션**: 현재 단계의 카드/버튼을 선택하면 의미 단위 이벤트(예: `ConfirmPrescription`, `PressPump`, `SelectLandmark`)를 발행한다.
2. **손 애니메이션**: Basic 액션 이벤트는 손 모델 애니메이션과 3D 오브젝트 상태 변화를 함께 트리거한다.
3. **Hard Hand 입력**: Hard Hand 모드가 ON일 때만 직접 손/팔 입력을 받는다. 입력은 `HandActionEvents`로 변환해 step controller에 전달한다.
4. **하이라이트**: 마우스 hover → 아웃라인 셰이더 노란색. 현재 단계와 무관한 오브젝트는 회색 아웃라인(약한 강도).
5. **선택**: 좌클릭 → 우측 상세 패널에 이름·설명·가능 액션 버튼.
6. **드래그**: Basic 모드에서는 최소화하고, 알콜솜 소독·주사기 경로 같은 연속 동작에서만 사용한다. Hard Hand 모드에서는 손 조작 자체가 연속 입력을 담당한다.
7. **순서 강제**: 현재 단계에 매칭되지 않는 상호작용 시도 → "지금은 {단계명}을 완료해야 합니다" 토스트.
8. **취소/되돌리기**: ESC로 현재 단계 취소 가능(감점 없이 재시작), 단 체크리스트 초기화.

## 6. 접근성 및 설정

| 항목 | 기본 | 옵션 |
|---|---|---|
| 자막 | ON | ON/OFF, 폰트 크기 2단계(Normal/Large) |
| 색맹 모드 | OFF | Deuteranopia 팔레트 |
| 마스터 볼륨 / BGM / SFX | 80 / 60 / 100 | 0~100 |
| 마우스 감도 | 중 | 저/중/고 |
| 시뮬레이션 모드 | Basic Simulation | Basic Simulation / Hard Hand Simulator |
| 언어 | 한국어 | (MVP는 한국어 단일, UI 훅만 남겨둠) |

## 7. 사운드 디자인 요점

- UI 클릭: 가벼운 soft tick.
- 정답/오답 피드백: 단음 ding / muted buzz.
- 앰비언트: 병실 룸톤 (루프, -24dB).
- 주사 장면: 실제 감상이 불쾌하지 않도록 절제 — "약한 쉭" 톤.
- 심박 타이밍 없음(긴장감 과다 유발 방지).
