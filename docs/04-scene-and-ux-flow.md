# 04. 씬 구성 및 UX 플로우

## 1. 씬 목록

| 씬 | 역할 | 활성 시스템 |
|---|---|---|
| `MainMenu` | 시작 화면, 시나리오 선택(MVP는 1개), 설정 | UI only |
| `Briefing` | 상황·처방 확인, 학습 목표 안내, "시작" 버튼 | UI + 짧은 나레이션 |
| `Simulation_IMInjection` | 메인 플레이 씬 (처치실) | Runner + Interaction + 3D 환경 |
| `Debriefing` | 결과 리포트, 재도전 | UI + ScoreTracker 읽기 |

씬 전환은 `SceneLoader`(싱글톤 or 정적 서비스)가 페이드 인/아웃과 함께 처리.

## 2. 글로벌 UX 플로우

```
[MainMenu]
   │  시작
   ▼
[Briefing]  ─(ESC: 메뉴)
   │  확인·시작
   ▼
[Simulation_IMInjection]
   │  절차 완료(11단계) 또는 "포기"
   ▼
[Debriefing]
   │  재도전 ──► [Briefing] (진행 상태 리셋)
   │  메인으로 ──► [MainMenu]
```

## 4. 카메라 설계 (Simulation 씬)

- **기본 카메라 (Cinemachine Virtual Camera #1)**: 처치실 고정 3인칭. 트레이·환자·학생 포지션이 모두 보이는 앵글.
- **상호작용 줌 카메라 (#2)**: 선택된 3D 도구/부위로 클로즈업. `CinemachineBrain`이 부드럽게 블렌드.
- **환자 부위 카메라 (#3)**: 부위 선정·랜드마크 단계에서 환자 해당 부위로 전환.
- 우클릭 드래그로 현재 가상 카메라의 `POV` 오빗(±15°)만 허용 — 자유시점은 UX 복잡도 증가로 제외.

## 5. 인터랙션 규칙

1. **하이라이트**: 마우스 hover → 아웃라인 셰이더 노란색. 현재 단계와 무관한 오브젝트는 회색 아웃라인(약한 강도).
2. **선택**: 좌클릭 → 우측 상세 패널에 이름·설명·가능 액션 버튼.
3. **드래그**: 알콜솜 소독·주사기 경로 같은 연속 동작에서만 사용. 시작점에서 누르고 드래그, 릴리즈 시 검증.
4. **순서 강제**: 현재 단계에 매칭되지 않는 상호작용 시도 → "지금은 {단계명}을 완료해야 합니다" 토스트.
5. **취소/되돌리기**: ESC로 현재 단계 취소 가능(감점 없이 재시작), 단 체크리스트 초기화.

## 6. 접근성 및 설정

| 항목 | 기본 | 옵션 |
|---|---|---|
| 자막 | ON | ON/OFF, 폰트 크기 2단계(Normal/Large) |
| 색맹 모드 | OFF | Deuteranopia 팔레트 |
| 마스터 볼륨 / BGM / SFX | 80 / 60 / 100 | 0~100 |
| 마우스 감도 | 중 | 저/중/고 |
| 언어 | 한국어 | (MVP는 한국어 단일, UI 훅만 남겨둠) |

## 7. 사운드 디자인 요점

- UI 클릭: 가벼운 soft tick.
- 정답/오답 피드백: 단음 ding / muted buzz.
- 앰비언트: 병실 룸톤 (루프, -24dB).
- 주사 장면: 실제 감상이 불쾌하지 않도록 절제 — "약한 쉭" 톤.
- 심박 타이밍 없음(긴장감 과다 유발 방지).
